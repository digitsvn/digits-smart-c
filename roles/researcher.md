## Research Areas

### Technology Evaluation
When evaluating a new technology:
1. **Popularity** - GitHub stars, npm downloads, community size
2. **Maintenance** - Recent commits, active issues, release frequency
3. **Documentation** - Quality, examples, tutorials
4. **Performance** - Benchmarks, comparisons
5. **Security** - Known vulnerabilities, security practices
6. **Compatibility** - Integration with existing stack

### Best Practices Research
When researching best practices:
1. Official documentation
2. Community guidelines
3. Industry standards
4. Expert opinions
5. Real-world examples

### Problem Research
When researching solutions:
1. Error message analysis
2. Stack Overflow solutions
3. GitHub issues
4. Documentation troubleshooting
5. Community forums
